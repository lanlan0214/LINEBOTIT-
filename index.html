1 VERIFICATION PHASE

QR 掃碼驗證階段

使用者透過 LIFF App 掃描設備 QR Code，系統解析序號並建立草稿工單，完成身分綁定

📷

LIFF 掃碼啟動

  • 初始化 liff.init() 取得 Access Token
  • 調用相機 QrScanner 元件
  • 支援 URL 參數 ?itemSerialNo= 直接帶入
  • 未登入自動跳轉 liff.login()
🔍

QR 內容解析

  • 白名單主機驗證 ALLOWED_HOSTS
  • 移除零寬字元、正規化 URL
  • 大小寫不敏感參數擷取
  • URL 解析失敗時 Regex 後援
📝

草稿工單建立

  • 呼叫 /api/notify 傳送序號
  • createTicketDraftOnScan() 寫入 DB
  • 狀態初始化 status=draft, phase=draft
  • 自動產生 ticketNo 並綁定 userId
👤

使用者 Upsert

  • fetchUserProfileByUserId() 取檔案
  • 儲存 displayName / pictureUrl / language
  • follow / unfollow 事件自動處理
  • 防連點鎖定 acquireRestartLock()
2 AI CLASSIFICATION

AI 意圖辨識階段

使用者輸入訊息後，系統透過多層判斷機制辨識是否為報修需求，並分類問題類型

🔤

規則層快篩

  • Regex 關鍵字比對：黑畫面、無法開機、斷線...
  • selectRepairPhrases() 預篩報修語句
  • 支援近 50 種常見報修關鍵詞
  • 通過快篩可跳過 AI 呼叫，降低延遲
🤖

OpenAI GPT 分類

  • aiClassifyRepairIntent() JSON 回傳
  • 輸出 is_repair / confidence / category
  • Prompt 版本由 DB prompts 表管理
  • Temperature 0.1，確保穩定輸出
⚡

快取與效能

  • Cache Key 綁定 prompt hash + model
  • withTTL() TTL 快取機制
  • 相同訊息不重複呼叫 API
  • 回傳 quick_replies 供使用者快速選擇
📊

分類結果處理

  • is_repair = true → 進入排障 / 收集流程
  • is_repair = false → 轉 RAG 知識庫回覆
  • 模糊症狀 → 釐清提問 promptClarifySymptom()
  • 完整日誌寫入 ticket_logs
3 AI TRIAGE & TROUBLESHOOTING

AI 自助排障階段

針對常見 IT 問題，AI 自動生成最多 6 步排障指引，嘗試在建單前協助使用者自行解決

🔧

通用排障引擎

  • aiTroubleshootGeneric() 通用問題
  • GPT 生成結構化 steps 陣列（≤ 6 步）
  • 自動編號輸出，控制在 800 字以內
  • 附帶追問：「請問問題是否已解決？」
💻

應用程式專用排障

  • looksLikeAppLaunchIssue() 偵測啟動類問題
  • aiTroubleshootApp() 專屬 Prompt
  • 情境預設：Windows / Office / 瀏覽器環境
  • Outlook、Teams、VPN、ERP 等常見應用
✅

解決狀態追蹤

  • Quick Reply：「已解決 / 未解決」
  • isYesReply() / isNoReply() 語意判斷
  • 已解決 → 結案 resetCase()
  • 未解決 → 進入資料收集階段
📚

RAG 知識庫輔助

  • Milvus 向量資料庫儲存歷史案例
  • answerByCompanyRag() 相似度檢索
  • COSINE 相似度 + HNSW 索引
  • 低於 RAG_MIN_SCORE 門檻不回覆
4 DATA COLLECTION

報修資訊收集階段

系統透過對話式 UX 逐步收集五大必填欄位，搭配 AI 欄位擷取實現一次輸入多欄位解析

📋

五大必填欄位

  • 客戶名稱 — 公司 / 機構名稱
  • 客戶單位 — 分行 / 部門
  • 聯絡人 — 姓名（≥ 2 字，無數字）
  • 連絡方式 — 電話 / LINE ID
  • 服務地址 — 完整地址（≥ 5 字）
🤖

AI 欄位擷取

  • aiExtractRepairInfo() 一次解析多欄位
  • 從自然語言中萃取結構化 JSON
  • 支援 call_reason / note / type 分類
  • Temperature 0.1，最大 1500 tokens
✏️

驗證規則

  • isPhone() — 台灣手機/市話/國際格式/分機
  • isPhoneOrLine() — 支援 LINE ID @xxx
  • isName() — 至少 2 字、不含數字
  • isAddress() — 至少 5 字
🔄

對話狀態機

  • SupportState 追蹤 phase / expect
  • hasAllRequiredV2() 檢查完整性
  • 歷史聯絡人自動帶入 getLastContactForUser()
  • 逾時自動清除驗證，要求重新掃碼
5 CONFIRMATION & SUBMISSION

確認與提交階段

顯示完整報修摘要供使用者確認，支援逐欄修改 / 全部修改 / 補充內容，確認後正式建立工單

📄

確認卡片生成

  • promptConfirmInfo() 產出摘要文字
  • 顯示：單號 / 症狀 / 客戶 / 聯絡人 / 地址
  • 包含隱私權聲明告知
  • Quick Reply 按鈕提供修改選項
✏️

互動修改選項

  • buildConfirmQuickReply() 動態生成按鈕
  • 支援修改 / 新增個別欄位
  • 「補充內容」欄位追加資訊
  • 「全部修改」重新收集所有欄位
🚀

工單正式建立

  • Postback action: 'submit' 觸發
  • createTicketCore() 寫入或更新 DB
  • ticketNo 格式：LR-YYYYMMDD-HHmmss-xxx
  • 狀態更新 draft → open
🔚

流程結束

  • 回覆確認訊息：案件已收到，轉交客服
  • resetCase() 清除所有暫存狀態
  • 設 mustRescan = true 下次需重新掃碼
  • Flex Carousel 卡片收尾 getRenderedPayload()
6 ENCRYPTION & STORAGE

資料加密與安全儲存

所有個人識別資訊 (PII) 採用 AES-256-GCM 欄位層級加密，確保靜態資料安全

🔐

AES-256-GCM 加密

  • 金鑰來源 APP_ENC_KEY_B64 (Base64 32 bytes)
  • 隨機 IV 12 bytes + Auth Tag 16 bytes
  • 版本前綴 ENC.v1. 支援金鑰輪替
  • seal() 加密 / open() 解密
🛡️

PII 欄位保護

  • custName / custBranch → 加密儲存
  • contactName / contactPhone → 加密儲存
  • contactAddress → 加密儲存
  • 防止重複加密 isEncrypted() 檢查
🗄️

MSSQL 資料庫

  • repair_tickets — 主工單表
  • ticket_logs — 完整對話日誌
  • flex_carousels — Flex Message 模板
  • Knex.js 管理 20+ 筆 Migration
⚙️

工單狀態管理

  • Status: draft → open → in_progress → closed
  • Phase: 10 種對話階段精準追蹤
  • goPhase() 同步寫入 phase + log
  • 支援 voided / voided_for_restart 作廢流程
7 NOTIFICATION & INTELLIGENCE

通知派工與智慧回覆

工單建立後自動發送 Email 通知客服人員，RAG 引擎持續提供歷史案例輔助回覆

📧

Email 通知系統

  • Nodemailer SMTP 郵件發送
  • SMTP 設定由 DB SystemParameter 動態取得
  • 通知內容：單號、客戶、聯絡人、症狀描述
  • 寄送成功/失敗完整 Log 記錄
🧬

RAG 向量檢索

  • Milvus 向量資料庫 + HNSW 索引
  • OpenAI Embeddings 語意向量化
  • COSINE 相似度匹配歷史案例
  • 公司層級隔離 company_key
📖

RAG 回覆生成

  • 檢索 Top-K 相似案例作為 Context
  • GPT 根據案例生成專業回覆
  • 不足以判斷時提出追問 + 通用排查
  • 回覆限制 8 行以內，繁體中文輸出
🔄

資料同步管線

  • rag-sync-hsis.ts 同步 HSIS 案例
  • SHA-1 穩定 ID 避免重複寫入
  • Collection 自動建立 + 欄位定義
  • 支援增量同步、批次匯入
